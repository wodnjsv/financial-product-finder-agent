# 07. 유동성 레짐 레이어 — 측정·분류·게이트

> 상태: 설계 초안 (2026-07-07) · 담당: 매크로·금리 애널리스트(`MC`, [02 §12](02-sector-analysts.md)) — 새 애널리스트를 추가하지 않고 MC의 임무를 확장한다.

## 1. 원리: 유동성은 방향, 펀더멘털은 선택

가격은 한계에서 결정된다 — 단기 방향은 자금·심리(유동성)가, 그 흐름 안에서의 상대 우열은 펀더멘털이 정한다. 둘은 서로 다른 질문에 답하며 동시에 성립한다:

| 층 | 답하는 질문 | 담당 |
|---|---|---|
| **유동성 레짐 (베타)** | 시장 전체가 어느 쪽으로 가는가 — 방향·노출·타이밍 | 이 문서 (MC 애널리스트) |
| **펀더멘털 (알파)** | 그 흐름 안에서 어떤 상품이 상대적으로 나은가 | 11개 섹터 애널리스트 ([02](02-sector-analysts.md)) |

최종 추천 = **섹터 분석(선택) × 유동성 레짐(방향)의 곱**. 위험회피 국면이면 반도체 애널리스트가 아무리 강세여도 추천은 방어 쪽으로 틀어지고, 위험선호 국면이면 펀더멘털 확신이 더 공격적으로 표현된다.

## 2. 불변 원칙: 측정(nowcasting)만, 예측(forecasting)은 하지 않는다

- **하는 것**: "지금 완화인가 긴축인가, 위험선호인가 회피인가, 자금이 들어오나 나가나"를 데이터로 분류한다. 이것은 가능하고 값이 크다.
- **하지 않는 것**: "연준이 다음에 어떻게 할지, 자금이 어디로 튈지"를 맞히려 들지 않는다. 정책·포지셔닝 예측은 매크로 헤지펀드도 자주 지는 게임이고, 넓은 레짐(완화/긴축)은 이미 상당히 가격에 반영돼 있다.
- 레짐 분류의 가치는 예측 알파가 아니라 **추천의 위험 표현을 현재 국면에 정합시키는 것**이다. "지금 썰물이다"를 아는 것만으로 방어 틸트를 걸 수 있으면 충분하다.
- **반사성 주의**: 유동성→펀더멘털(싼 돈은 리파이낸싱·투자를 살리고 긴축은 부도를 늘림), 펀더멘털→자금 유입의 양방향 피드백이 있다. 깔끔한 일방향 예측 변수로 취급하지 않는다.

## 3. 유동성의 4개 층

층마다 데이터 소스·시간축·신호 성격이 다르다. **섞으면 노이즈가 되므로 반드시 구분해서 측정한다.**

| 층 | 내용 | 시간축 | 핵심 지표 (소스) |
|---|---|---|---|
| **L1. 매크로 순유동성** | 중앙은행 대차대조표, QT/QE, 정부 현금 계정 | 가장 느림 (월~분기) | Fed 총자산 `WALCL`, 역레포 `RRPONTSYD`, 재무부 TGA `WTREGEN` (FRED) → 미국 순유동성 = WALCL − RRP − TGA · 한은 기준금리·통화량 M2 (ECOS) |
| **L2. 금융환경** | 위험선호의 온도계 | 주 단위 | 회사채 스프레드 = 회사채(AA-·BBB-) − 국고채 (ECOS), `VIXCLS`, 달러지수 `DTWEXBGS`, `T10Y2Y` (FRED) |
| **L3. 자금 흐름** | 펀드·ETF 유출입, 외국인 수급 | 일~주 | 국내 ETF 자금 유출입, **KRX 투자자별 매매동향(외국인·기관·개인)**, 외국인 주식 보유·순매수 (KIS API/KRX) |
| **L4. 미시 유동성** | 호가·체결 수준의 수급 | 일 중~일 | 거래대금, 개별 상품 거래량·괴리율 (KIS API) — v0에서는 상품별 유동성 필터로만 사용 |

### 한국 시장의 구조적 엣지: L3

한국은 **투자자별 매매동향(외국인/기관/개인)이 KRX를 통해 매일 투명하게 공개**되고, 코스피는 외국인 수급 민감도가 매우 크다. 미국에서 기관 포지셔닝이 불투명·고비용·지연 데이터인 것과 대조적이다. 넓은 레짐(L1·L2)은 모두가 보는 신호라 엣지가 얇지만, **세분화된 L3(특히 외국인 흐름)은 덜 효율적으로 반영되는 영역**이라 이 시스템이 실제로 차별화할 수 있는 지점이다.

## 4. 레짐 분류 체계 (v0)

### 4.1 두 축, 이산 상태

```
축 1 — 유동성:   완화 | 중립 | 긴축     (L1 중심, L2 보조)
축 2 — 위험선호: risk-on | neutral | risk-off   (L2 중심, L3 보조)
```

v0 산출 방법: 각 층의 지표를 z-score로 표준화 → 층별 합성 점수 → 임계값으로 이산 상태 분류. **규칙 기반, LLM 아님.** MC 애널리스트는 분류 결과에 해석 노트(근거·특이사항)를 붙인다.

- 상태 전환에는 히스테리시스(완충 구간)를 둬서 경계 근처에서 매일 뒤집히는 것을 방지한다.
- 모든 레짐 판정은 `RegimeState { axis1, axis2, as_of, 입력 지표 스냅샷, 해석 노트 }`로 **시점 기록** — 레짐 분류 자체도 [05](05-analyst-structure.md)의 평가 대상이다 (사후에 "그때 risk-off 판정이 맞았나"를 검증).

### 4.2 그래프 표현

`RegimeState`는 그래프의 노드로 저장하고 (`AnalystView`와 동일한 시간성 규약), 현재 유효 레짐은 랭킹 엔진이 매 질의마다 조회한다.

## 5. 레짐 → 랭킹 게이트

레짐은 개별 상품 점수를 직접 매기지 않는다. **랭킹 스펙의 가중치 프로파일과 애널리스트 뷰의 표현 강도를 조절**한다 ([06](06-query-planner.md)의 `ranking_spec`에 곱해지는 조절기).

### 5.1 게이트 규칙 (v0)

| 레짐 | 뷰 가중 계수 | 랭킹 틸트 |
|---|---|---|
| risk-on + 완화 | ×1.0 (뷰를 온전히 표현) | 테마 노출도 가중 ↑ |
| neutral | ×0.7 | 기본 프로파일 |
| risk-off | ×0.4 (강세 뷰의 표현 축소) | 저베타·단기 듀레이션·배당·현금성 가중 ↑, 고베타 테마 가중 ↓ |
| risk-off + 긴축 | ×0.25 | 위와 동일 + 신용 하위등급 soft 필터 강화 |

- 게이트는 **비대칭**이다: risk-off에서 약세 뷰(위험 회피 방향)는 축소하지 않는다. 줄이는 것은 위험을 늘리는 방향의 표현뿐이다.
- 사용자가 명시적으로 공격형·특정 테마를 요구하면 게이트는 **추천을 막지 않고 고지로 전환**한다: "현재 위험회피 국면이라 변동성이 클 수 있습니다" — 사용자 의도([06] 명시적 제약)가 시스템 판단보다 우선한다는 원칙의 연장.
- 게이트 적용 내역은 `relaxation_log`처럼 explanation trace에 기록되어 응답 합성이 반드시 설명한다.

### 5.2 랭킹 3단 → 4단

[01 §3-③](01-architecture.md)의 점수 조합이 다음으로 확장된다:

```
1. 하드 필터 (통과/탈락)
2. 그래프 노출도 점수
3. 애널리스트 뷰 가중        ← 여기에 레짐 게이트 계수가 곱해짐
4. 레짐 틸트                 ← 랭킹 스펙 가중치 프로파일 조절
```

## 6. MC 애널리스트의 확장된 산출물

기존 커브 뷰·듀레이션 스탠스·크레딧 스탠스·환율 뷰([02 §12](02-sector-analysts.md))에 추가:

- **레짐 판정** (일 1회 + 임계 이벤트 시): §4의 RegimeState
- **레짐 해석 노트**: 분류 근거, 전환 임박 신호, 층간 불일치(예: L1 완화인데 L3 외국인 유출) — 불일치 자체가 유용한 정보
- **섹터 애널리스트 배포**: 레짐 전환 시 전 애널리스트에게 컨텍스트로 배포 (뷰 갱신 트리거)

## 7. 엣지의 위치 (겸손한 요약)

이 레이어의 기대 가치는 순서대로:
1. **레짐 정합 추천** — 국면에 맞는 위험 표현 (가장 확실한 가치)
2. **한국 수급 데이터(L3)의 세분화 활용** — 덜 효율적인 신호
3. 레짐 전환의 조기 포착 — 가능하면 좋지만 기대치를 낮게

"유동성 지표 하나로 시장 예측"은 목표가 아니다.

## 8. Phase별 도입

| Phase | 범위 |
|---|---|
| 1 | 데이터 수집만: L1·L2 시리즈 (FRED·ECOS 배치에 포함), L3 KRX 수급 수집 시작. 레짐 분류·게이트 없음 |
| 2 | 규칙 기반 레짐 분류(z-score 합성) 가동 + 시점 기록 시작. 게이트는 **로그만 하고 미적용** (분류 품질 검증 기간) |
| 3 | 랭킹 게이트 실적용 + 레짐 판정의 사후 검증을 [05](05-analyst-structure.md) 평가 파이프라인에 통합 |
