# 04. 데이터 수집 계획

> 상태: 설계 초안 (2026-07-06) · 보유 API 키: FRED, ECOS, 토스증권, 한국투자증권(KIS)

## 1. 필요 데이터 인벤토리 → 소스 매핑

시스템이 요구하는 데이터를 먼저 정의하고, 보유/미보유 소스에 매핑한다.

| # | 데이터 | 용도 | 소스 | 보유 여부 |
|---|---|---|---|---|
| D1 | 상품 마스터 — ETF (종목명, ISIN, 기초지수, 보수, 상장일, AUM) | 하드필터, 그래프 Product 노드 | KIS API + KRX 정보데이터시스템 | 부분 보유 |
| D2 | ETF 구성종목(PDF)과 비중 | 그래프 HOLDS 엣지 — **노출도 랭킹의 심장** | KIS(구성종목 조회) / KRX / 운용사 공시 | 부분 보유 |
| D3 | 상품 마스터 — 채권 (발행정보, 만기, 표면금리, 신용등급) | 채권 추천 | **미보유** → 금투협 채권정보센터, SEIBro | ✗ |
| D4 | 채권 시가평가·수익률 | 채권 랭킹, 듀레이션 계산 | **미보유** → 금투협(KOFIA BIS) | ✗ |
| D5 | 상품 마스터 — 공모펀드 (기준가, 보수, 유형) | 펀드 추천 | **미보유** → 금투협 전자공시/펀드다모아 | ✗ |
| D6 | 주식·ETF 시세/NAV (일별) | 시계열, 성과 평가 | KIS API, 토스증권 API | ✓ |
| D7 | 국내 매크로 (기준금리, 국고채 커브, 회사채 수익률, CPI, 환율, 가계신용, 소비자심리) | 매크로 애널리스트, AFFECTS 트리거 | ECOS | ✓ |
| D8 | 글로벌 매크로 (연준, 미 국채, 유가, 달러지수, VIX, 미국 CPI·소매판매) | 매크로 애널리스트 | FRED | ✓ |
| D9 | 기업 공시 (사업보고서, 수주, 임상, 계약) | Layer 2 엣지 grounding, 벡터 저장소 | **미보유** → DART OpenAPI (무료 키 즉시 발급) | ✗ |
| D10 | 기업 기본정보·업종분류 | Company 노드, OPERATES_IN 근사 | KIS(종목정보) + DART + KRX | 부분 보유 |
| D11 | 산업 지표 (반도체 수출, 품목별 수출입, 자동차 판매 등) | 섹터 애널리스트 모니터링 | **미보유** → 관세청 무역통계, 공공데이터포털 | ✗ |
| D12 | 뉴스·리포트 텍스트 | 벡터 저장소, 뷰 근거 | **미보유** → 후순위 (Phase 2+) | ✗ |

## 2. 보유 소스 상세

### ECOS (한국은행 경제통계시스템) — 매크로의 주 소스
- 기준금리, 국고채 수익률(전 만기), 회사채 AA-/BBB- 수익률 → `MC.rates`, `MC.credit`
- 원/달러 환율, CPI, 가계신용, 소비자심리지수(CCSI), 건설수주액
- 수집: REST API, **일 1회 배치**면 충분 (통계별 공표 주기 상이 — 시리즈별 스케줄 테이블 관리)

### FRED — 글로벌 매크로의 주 소스
- `DFF`, `DGS2`, `DGS10`, `T10Y2Y`, `CPIAUCSL`, `DCOILWTICO`, `DCOILBRENTEU`, `DHHNGSP`, `DTWEXBGS`, `VIXCLS`, `RSAFS`, `UMCSENT`
- 수집: REST API, 일 1회 배치

### 한국투자증권 KIS OpenAPI — 상품·시세의 주 소스
- 국내 주식/ETF 현재가·일별 시세, 종목 기본정보
- ETF 구성종목 조회 (NAV, 괴리율 포함) → **D2의 1차 소스**
- 국내 채권 일부 시세 조회 가능 (커버리지 검증 필요)
- 주의: 호출량 제한(rate limit) 있음 → 전 종목 순회 배치는 큐·백오프 설계 필요

### 토스증권 API
- 시세·종목정보 보조 소스로 배치. **공개 API의 제공 범위·약관(데이터 저장 허용 여부)을 구현 전 확인 필요** — 확인 결과에 따라 KIS 단일 소스로 통합될 수 있다.

## 3. 미보유 → 추가 확보 계획 (전부 무료)

| 소스 | 확보 방법 | 채우는 데이터 |
|---|---|---|
| **DART OpenAPI** | 금융감독원, 무료 키 발급 | D9 공시 (Layer 2 grounding의 핵심), D10 기업정보 |
| **KRX 정보데이터시스템** | 웹 데이터 (CSV 다운로드/스크레이핑) | D1 ETF 전체 목록, D2 PDF 교차검증, 업종분류 |
| **금융투자협회 채권정보센터** | 웹/OpenAPI | D3, D4 채권 발행정보·시가평가·등급별 수익률 |
| **SEIBro (예탁결제원)** | 웹/OpenAPI | D3 채권 상세, D5 펀드 보조 |
| **금투협 전자공시·펀드다모아** | 웹 | D5 공모펀드 마스터·기준가·보수 |
| **관세청 무역통계 / 공공데이터포털** | OpenAPI 키 발급 | D11 반도체 수출액, 품목별 수출입 (IT·소재·필수소비재 애널리스트용) |

## 4. 파이프라인 구조

```
[스케줄러 (크론)]
   │
   ├─ daily 06:30  ECOS·FRED 시리즈 → 시계열 저장소
   ├─ daily 08:00  KIS: ETF NAV·시세, 구성종목 스냅샷 → 시계열 + 그래프 HOLDS 갱신
   │                (HOLDS는 diff 감지 → 변경분만 valid_to 마감 + 신규 엣지)
   ├─ daily 18:00  DART 신규 공시 수집 → 벡터 저장소 적재 (+ 애널리스트 트리거 큐)
   ├─ weekly       채권 마스터·시가평가, 펀드 기준가·마스터 갱신
   └─ monthly      상품 마스터 전수 정합성 점검 (상장폐지·신규상장 반영)
```

공통 규칙:
- **원본 보존**: API 응답 원본(raw)을 그대로 적재 → 가공 테이블은 언제든 재생성 가능 (스키마 실수 복구 비용 최소화)
- **as_of 각인**: 모든 레코드에 수집 시점과 데이터 기준일을 분리 기록 (point-in-time 원칙)
- **이상 감지**: 결측·급변(전일 대비 임계 초과) 시 적재 보류 + 알림

## 5. Phase 1 수집 우선순위

Phase 1(하드필터 + 그래프 노출도 랭킹)에 필요한 최소 집합:

1. **D1+D2 (ETF 마스터 + 구성종목)** — KIS API로 시작. 이게 없으면 그래프가 비어 있다.
2. **D10 (기업-업종 매핑)** — OPERATES_IN 근사의 재료.
3. **D7+D8 (매크로)** — 배치가 단순해서 병행 구축 비용이 낮고, Phase 2 매크로 애널리스트의 재료가 미리 쌓인다.
4. 채권(D3·D4)·펀드(D5)는 **ETF 파이프라인 검증 후** 확장 — 소스가 전부 다르므로 수직으로 하나씩.

## 6. 리스크·확인 필요 사항

- **토스증권 API**: 공개 범위·데이터 저장 약관 미확인. 확인 전까지 설계는 KIS 단일 소스 가정.
- **KIS 호출량 제한**: 전 ETF(약 900종) 구성종목 일일 순회가 한도 내인지 검증 필요. 초과 시 KRX 파일 다운로드로 대체.
- **펀드 데이터 지연**: 공모펀드 기준가·포트폴리오는 공시 주기가 길다(분기) → 펀드의 그래프 노출도는 ETF보다 해상도가 낮음을 랭킹에서 명시(신뢰도 할인).
- **신용등급 데이터**: 금투협 커버리지 확인 필요. 부족 시 신평사(한신평·한기평·나신평) 공시 크롤링 검토.
